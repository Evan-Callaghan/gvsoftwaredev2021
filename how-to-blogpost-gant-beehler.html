How to write a blog post in html

<br> is not needed, you can use <p> tags

Creating a blog post in HTML only requires a few lines of HTML and since we want everyone’s blog post to look alike, it’s even easier. We would start this page of our website using establishing that we are going to use html by using the To make the code appear, use .&(gt) and .&(lt) without the parentheses for the greater than and less than signs: <!DOCTYPE html> and <html lang=en> .

Then we want to bound a header to our data, within the header we give a title so that the program has a title for the data about to imputed. When the data is outputted, it will not show the title. We then close the head and open the body; this is where a headline statement and blog information will go. Headlines have sizes 1 through 6, 1 being the biggest and 6 being the smallest. A(n) remove header statement, would be ideal to state what this page of our website entails. For an example, we set this up as if we are blogging our Goals for the semester. So, we named it Software development: Class goals.

Then, to separate each person’s goal statement, use another header statement, we could use anywhere between two and six. We suggest two because it’s smaller than the page headline but a little larger than the text so you can clearly see who’s post you’re reading.

Then for our main blog post, use a paragraph element, add in all the necessary information, and end the statement. Anytime we are ending a statement, we will use the original statement starter, ie. <h1> and add a backslash after the first arrowhead </h1>. The next person who wants to add a post would just insert a new header statement for their name and a paragraph element under the previous student! We added in an example as if Morgan blogged her Class goals and then Evan did after!

Most of the Student’s goal statements are pretty long and were broken down in paragraphs so we decided to do a little cleaning and add a break statement after each paragraph ended so it would send the next line of the inputted blog information of the paragraph elements to the line below. There are other things we can do to clean up the statements, but that will come from other students!

After all the blogs are written in, we would close the body and html opening statement.

Because the <pre> tag takes and writes your code as you put it in, the code will spill over as there is no real "break" in your statements. You will need to add your breaks in by using the enter button on your computer
